فهم مسؤولياتك عند استخدام Qiskit Runtime

راجع الأقسام التالية لمعرفة المسؤوليات المحددة التي تقع عليك وعلى IBM® عند استخدام Qiskit Runtime. للاطلاع على الشروط العامة للاستخدام، راجع الشروط والإشعارات الخاصة بـ IBM Cloud®.

إدارة الحوادث والعمليات​

تشمل إدارة الحوادث والعمليات مهام مثل المراقبة، وإدارة الأحداث، والتوافر العالي، وتحديد المشكلات، والاسترداد، والنسخ الاحتياطي الكامل والاسترداد.

المهمةمسؤوليات IBMمسؤولياتك
إدارة صحة البنية التحتية
  • توصيل الميزات الجديدة والتحديثات وإصلاحات الأخطاء باستمرار بطريقة شفافة حسب الحاجة.
  • إرسال إشعارات بالصيانة المجدولة ونشرات الأمان أو انقطاعات الخدمة غير المخططة.
  • ضبط التفضيلات لاستقبال رسائل البريد الإلكتروني بشأن إشعارات الخدمة.
  • مراقبة صفحة حالة IBM Cloud للاطلاع على الإعلانات العامة.
مراقبة صحة البنية التحتية والإشعار بها
  • معالجة جميع مشكلات بيئة البنية التحتية.
  • إخطار العملاء بالحوادث المعنية.
  • تقييم تأثير كل حادث مُبلَّغ عنه.
  • التواصل مع دعم IBM عند الحاجة.
الحفاظ على اتفاقية مستوى الخدمة للتوافر العالي في IBM Cloudتشغيل Qiskit Runtime وفقاً لـ اتفاقيات مستوى الخدمة العامة لـ IBM Cloud (SLAs).اتباع أفضل ممارسات التوافر العالي.

إدارة التغييرات​

تشمل إدارة التغييرات مهام مثل النشر، والتكوين، والترقيات، والتصحيح، وتغييرات التكوين، والحذف.

المهمةمسؤوليات IBMمسؤولياتك
ترقيات الإصدارات الرئيسيةتوفير الإمكانية والأدوات اللازمة لتحديثات الإصدارات الرئيسية.الانتقال إلى إصدار مدعوم قبل انتهاء فترة الدعم للإصدار القديم.
تغييرات إصدار واجهة برمجة التطبيقاتإصدار نسخ من REST APIs.التكامل مع إصدار REST APIs المطلوب.
التحديثات والإصلاحات والميزات الجديدة
  • تقديم تحديثات دورية وإصلاحات للأخطاء وميزات جديدة بشكل شفاف وفق نموذج التسليم المستمر.
  • نشر إشعارات بالتغييرات التي تؤثر على العملاء.
  • ضبط التفضيلات لاستقبال إشعارات البريد الإلكتروني ومراقبة صفحة حالة IBM Cloud للاطلاع على الإعلانات العامة.
  • قراءة الإشعارات التلقائية من النظام وإعلانات المنصة، واتخاذ الإجراء المناسب.

إدارة الهوية والوصول​

تشمل إدارة الهوية والوصول مهام مثل المصادقة، والترخيص، وإدارة سياسات التحكم في الوصول، والموافقة على الوصول ومنحه وإلغائه.

المهمةمسؤوليات IBMمسؤولياتك
التحكم في الهوية والوصولتوفير القدرة على إدارة الوصول إلى الموارد وتقييده عبر IBM Cloud.منح الوصول إلى الموارد وإلغاؤه وإدارته عبر Cloud Identity and Access Management (IAM).
مصادقة الخدمةضمان أن المستخدمين المصادق عليهم فقط هم من يمكنهم الوصول إلى خدمات Qiskit Runtime والموارد الخاصة بك.
  • حماية مفتاح API الخاص بك من الوصول غير المصرح به.
  • تدوير مفاتيح API وتحديثها وفقاً لمتطلبات سياسة الأمان الخاصة بك.
قابلية المراقبةالسماح بتكامل IBM Cloud Activity Tracker Event Routing لمراجعة إجراءات المستخدم.إعداد IBM Cloud Activity Tracker Event Routing أو وظائف أخرى لتتبع نشاط المستخدم.

الأمان والامتثال التنظيمي​

يشمل الأمان والامتثال التنظيمي مهام مثل تطبيق ضوابط الأمان وشهادات الامتثال.

المهمةمسؤوليات IBMمسؤولياتك
عام
  • الحفاظ على ضوابط تتناسب مع معايير الامتثال الصناعية المختلفة.
  • مراقبة وظائف الخدمة وعزلها واستردادها.
  • مراقبة صحة واجهات الخدمة والإبلاغ عنها.
التأكد من عدم تقديم بيانات شخصية أو بيانات خاضعة للتنظيم.
الثغرات الأمنيةالمراقبة للكشف عن مشكلات الثغرات الأمنية والامتثال الأمني.
  • البقاء على اطلاع بالثغرات المحتملة والمشكلات الأمنية عبر النشرات الأمنية التي تقدم إجراءات معالجة محتملة.
  • مراجعة قسم الأمان في مستودع Qiskit على GitHub. يتم نشر جميع الثغرات الأمنية هناك.
  • مراجعة موقع حالة IBM Cloud للاطلاع على الإعلانات والنشرات الأمنية حول الأحداث الرئيسية التي تؤثر على منصة IBM Cloud والبنية التحتية والخدمات الرئيسية.
التشفيرتشفير البيانات عند نقلها عبر أي شبكات عامة وداخل شبكة مركز البيانات الخاصة بالخدمة السحابية. تُستخدم طرق التشفير مثل HTTPS وTLS لحماية البيانات أثناء النقل.التأكد، عند الاقتضاء، من أن اتصالك مشفر من طرف إلى طرف.
الأمان
  • ضمان أمان البيانات على القرص والبيانات أثناء النقل داخل بنيتها التحتية.
  • تقديم خدمة آمنة تمتثل للمعايير الرئيسية.
تقييد وصول المستخدم إلى الموارد ومجموعات الموارد المناسبة.

التعافي من الكوارث​

المهمةمسؤوليات IBMمسؤولياتك
عاماستعادة بيئات التوفير والموارد في المناطق المتأثرة أو إعادة بنائها.
  • مراقبة حالات النسخ والموارد.
  • توفير نسخ جديدة من خدمة Qiskit Runtime في منطقة بديلة، إذا كانت متاحة.
استرداد الخدماتالاسترداد التلقائي لقدرات خدمة Qiskit Runtime واستعادتها إلى العمليات الطبيعية.إعادة إرسال أي مهام تأثرت عقب الاسترداد.
النسخ الاحتياطي لبيانات تكوين المستخدمIBM غير مسؤولة عن النسخ الاحتياطي لبيانات تكوين المستخدم.النسخ الاحتياطي لأي بيانات تكوين، حسب الحاجة.
النسخ الاحتياطي واسترداد أعباء العملIBM غير مسؤولة عن النسخ الاحتياطي أو استرداد بيانات أعباء عمل المستخدم.النسخ الاحتياطي واسترداد أعباء العمل الخاصة بك، حسب الحاجة.
